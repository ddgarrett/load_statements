## How Wells Fargo is wired today

Detection is **not** by friendly names like “Wells Fargo”; it’s by the **exact text of the first row** of the CSV. In `main.py`, a row matches if `",".join(line).startswith(key)` for some key in `func_lookup`:

```24:30:main.py
def test_line(tst:str,line:list[str]) -> bool:
    '''
        Return True if the string array 'line'
        starts with the same strings as 'tst'.
    '''
    l = ','.join(line)
    return l.startswith(tst)
```

So the lookup string must match the **beginning** of the header row (including order and spelling).

The Wells key and loader are in `load_txn.py`:

It only uses the **CSV filename**, not the header row or row contents.

In `load_wells_txn`, the account label is set from `fileName` (the basename of the file being processed):

```19:22:load_txn.py
def load_wells_txn(reader, fileName: str, hdr: str):
    account = "wellsfargo"
    if fileName.startswith("CreditCard"):
        account = "wfcc"
```

`main.py` passes `Path(fn).name` as `fileName`, so if the file is named like `CreditCardSomething.csv`, the stored account becomes `wfcc`; otherwise it stays `wellsfargo`.

There is **no** separate header string or loader for credit vs checking—the same `ld_wells` header matches both; only the filename decides `wellsfargo` vs `wfcc`.
